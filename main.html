Euclidean $(k,t)$-spanner

Richard Spence

CSC 544 Fall 2017

Given a set $V$ of points in the plane and a constant $t \ge 1$, we define a Euclidean t-spanner of $V$ to be a graph $G(V,E)$ spanning the points in $V$ such that for all $u, v \in V$, we have $d_G(u,v) \le t \cdot d(u,v)$ where $d_G()$ denotes the length of the shortest path in $G$ and $d()$ denotes the Euclidean distance between two points. Computing a $t$-spanner of $G$ with minimum total length is NP-hard for any $t > 1$ (Carmi et al., 2013).

A reasonable variant is what I call a $(k,t)$-spanner: given a set $V$, a constant $t \ge 1$, and an integer $k \ge 1$, a $(k,t)$-spanner is a $t$-spanner with the additional property that for all $u,v \in V$, there exists a path from $u$ to $v$ of length at most $t \cdot d(u,v)$ using no more than $k$ edges. Note that if $k = |V|-1$ then this reduces to the Euclidean $t$-spanner problem, and if $t$ is arbitrarily large then this reduces to the minimum spanning tree (MST) problem.

The user can input a value of $k \ge 1$ and $t \ge 1$. "Compute approx. t-spanner" will run a very simple heuristic and compare to the cost of the MST.
